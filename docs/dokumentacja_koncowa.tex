\section{Wprowadzenie i Cel Projektu}
% ============================================================
Niniejszy dokument stanowi dokumentację końcową projektu zrealizowanego w ramach
przedmiotu \textit{Systemy Ekspertowe}. Opisuje on gotowy, działający system
ekspertowy, którego zadaniem jest identyfikacja gatunku zwierzęcia na podstawie
jego cech fenotypowych (takich jak masa ciała, długość, długość życia, pokrycie
ciała) oraz środowiskowych (np. środowisko życia, kontynent występowania).

W odróżnieniu od prostych algorytmów decyzyjnych opartych na drzewach
decyzyjnych, zaimplementowany system wykorzystuje \textbf{wnioskowanie
dedukcyjne w środowisku niepewnym}. Łączy on ścisłą logikę języka Prolog
z mechanizmami logiki rozmytej (ang. \textit{fuzzy logic}) oraz teorią zbiorów
przybliżonych (ang. \textit{rough sets}), co pozwala na obsługę parametrów
o charakterze ciągłym i prowadzenie wnioskowania przybliżonego.

Całość systemu została zaimplementowana w języku \textbf{SWI-Prolog}.
Pomocniczy skrypt w języku Python posłużył do wygenerowania bazy wiedzy
z pliku danych w formacie CSV.

\subsection{Zakres funkcjonalny}
Zaimplementowany system umożliwia:
\begin{itemize}[noitemsep]
    \item klasyfikację zwierząt na podstawie cech podawanych interaktywnie przez
          użytkownika,
    \item wnioskowanie z niepewnością przy użyciu funkcji przynależności zbiorów
          rozmytych,
    \item dynamiczne sterowanie dialogiem w oparciu o moc dyskryminacyjną
          atrybutów (mierzoną entropią),
    \item analizę bazy wiedzy pod kątem reguł minimalnych, atrybutów niezbędnych
          (jądra) oraz reduktów.
\end{itemize}

\section{Modelowanie Wiedzy}
% ============================================================
Baza wiedzy nie jest prostym wykazem gatunków, lecz zbiorem faktów oddzielających
wiedzę o dziedzinie od logiki sterującej. Każde zwierzę opisane jest zestawem
atrybutów dyskretnych (deklarowanych predykatem \code{attribute/3}) oraz ciągłych
(predykat \code{continuous\_attribute/3}).

\subsection{Reprezentacja atrybutów rozmytych}
Jednym z kluczowych wyzwań jest obsługa parametrów rozmytych, takich jak masa
lub długość ciała. Zamiast stosować sztywne progi binarne przynależności (np.
\code{masa > 100 kg}), system wykorzystuje teorię zbiorów rozmytych. Zdefiniowane
są funkcje przynależności $\mu_A(x)$, co umożliwia przypisanie cechy do obiektu
z częściową pewnością.

\subsubsection{Typy funkcji przynależności}
\paragraph{Funkcja trapezoidalna}
Stosowana dla atrybutów posiadających przedział pełnej przynależności i częściową
przynależność na obu jego końcach. Oznaczamy ją jako $\mathrm{Trapez}(a,b,c,d)$.
\begin{equation}
\mu_T(x;a,b,c,d)=
\begin{cases}
0 & x \le a\\[2pt]
\dfrac{x-a}{b-a} & a < x \le b\\[6pt]
1 & b < x \le c\\[2pt]
\dfrac{d-x}{d-c} & c < x \le d\\[6pt]
0 & d < x
\end{cases}
\end{equation}

\paragraph{Funkcje L i R-kształtne}
Stosowane dla atrybutów, które dla wartości mniejszych (większych) od wartości
granicznej mają pełną przynależność, a na krańcu przynależność częściową.
Oznaczamy je jako $\mathrm{L\text{-}kszta\l tna}(a,b)$ oraz
$\mathrm{R\text{-}kszta\l tna}(a,b)$.
\begin{equation}
\mu_L(x;a,b)=
\begin{cases}
1 & x \le a\\[2pt]
\dfrac{b-x}{b-a} & a < x \le b\\[6pt]
0 & b < x
\end{cases}
\qquad
\mu_R(x;a,b)=
\begin{cases}
0 & x \le a\\[2pt]
\dfrac{x-a}{b-a} & a < x \le b\\[6pt]
1 & b < x
\end{cases}
\end{equation}

W implementacji funkcje te zostały zrealizowane bezpośrednio w pliku
\code{fuzzy.pl}:

\begin{lstlisting}[language=Prolog,caption={Funkcje przynależności (\code{fuzzy.pl})}]
trapez(X, A, B, C, D, Val) :-
    ( X =< A -> Val = 0.0
    ; X > A, X =< B -> Val is (X - A) / (B - A)
    ; X > B, X =< C -> Val = 1.0
    ; X > C, X =< D -> Val is (D - X) / (D - C)
    ; Val = 0.0
    ).

l_ksztaltna(X, A, B, Val) :-
    ( X =< A -> Val = 1.0
    ; X > A, X =< B -> Val is (B - X) / (B - A)
    ; Val = 0.0
    ).

r_ksztaltna(X, A, B, Val) :-
    ( X =< A -> Val = 0.0
    ; X > A, X =< B -> Val is (X - A) / (B - A)
    ; Val = 1.0
    ).
\end{lstlisting}

\subsection{Implementacja bazy wiedzy}
Baza wiedzy zawiera \textbf{100 gatunków} zwierząt. Aby uniknąć ręcznego
wpisywania faktów, dane przygotowano w pliku \code{zwierzeta\_atrybuty.csv},
a plik \code{kb.pl} generowany jest automatycznie skryptem \code{gen\_kb.py}.
Przykładowy fragment wygenerowanej bazy:

\begin{lstlisting}[language=Prolog,caption={Fragment bazy wiedzy (\code{kb.pl})}]
animal('lew').
attribute('lew', dieta, 'miesozerne').
attribute('lew', srodowisko, 'ladowe').
attribute('lew', skrzydla, 'nie').
attribute('lew', nogi, 'cztery').
attribute('lew', gromada, 'ssaki').
attribute('lew', kontynent, 'afryka').
% ... pozostale atrybuty dyskretne ...
continuous_attribute('lew', ciezar, 190.0).
continuous_attribute('lew', dlugosc, 250.0).
continuous_attribute('lew', dlugosc_zycia, 12.0).
\end{lstlisting}

Baza udostępnia tylko fakty -- nie zawiera żadnej logiki sterującej ani statycznie
zapisanych konkluzji, co realizuje wymóg separacji warstw (rozdział~\ref{sec:arch}).

\section{Mechanizm Wnioskowania i Interakcja z Użytkownikiem}
% ============================================================
System nie realizuje z góry ustalonej, statycznej sekwencji pytań. Przebieg
dialogu jest dynamicznym wynikiem procesu poszukiwania rozwiązania. Logika
dialogu zaimplementowana jest w pliku \code{ui.pl}, a sterowana jest pętlą
\code{ask\_loop/2}, która utrzymuje aktualny zbiór możliwych hipotez
(\textit{PossibleAnimals}) oraz zbiór jeszcze niewykorzystanych atrybutów.

\subsection{Dynamiczna selekcja pytań}
Interfejs na bieżąco analizuje aktualną przestrzeń hipotez i wybiera do zadania
pytanie o atrybut o \textbf{najwyższej mocy dyskryminacyjnej}. Moc ta jest
mierzona za pomocą \textbf{entropii} rozkładu wartości atrybutu wśród
rozważanych zwierząt -- wybierany jest atrybut o najniższej entropii warunkowej,
czyli ten, który najsilniej zawęża zbiór hipotez. Atrybuty o jednej wartości
(bezużyteczne dla rozróżnienia) są pomijane.

\begin{equation}
H(a) = -\sum_{v \in V_a} p(v)\,\log p(v)
\end{equation}

\begin{lstlisting}[language=Prolog,caption={Selekcja atrybutu na podstawie entropii (\code{ui.pl})}]
best_attribute_to_ask(Attrs, Animals, BestAttr) :-
    findall(Score-Attr, (
        member(Attr, Attrs),
        evaluate_attribute(Attr, Animals, Score)
    ), Scores),
    ( Scores = [] -> BestAttr = none
    ; sort(1, @=<, Scores, [MinScore-BestAttr|_]), % minimalizujemy entropie
      MinScore < 999 -> true
    ; BestAttr = none
    ).
\end{lstlisting}

\subsection{Kontekstowość dialogu}
Zgromadzone dotychczas informacje są zawsze uwzględniane przy wyborze kolejnego
pytania -- po każdej odpowiedzi zbiór hipotez jest filtrowany
(\code{filter\_animals/4}), a kolejne pytanie dobierane jest już do nowej,
zawężonej przestrzeni. Dzięki temu pytania o atrybuty, które straciły moc
rozróżniającą, nie są zadawane.

\subsection{Wielokrotny wybór}
System dopuszcza odpowiedzi wielokrotnego wyboru wszędzie tam, gdzie wynika to
z natury pytania. Użytkownik może podać kilka wartości oddzielonych przecinkami
(np. \code{tak,nie}); są one wówczas obsługiwane przez
\code{filter\_animals\_multi/4}, który zachowuje zwierzęta pasujące do
\textit{którejkolwiek} ze wskazanych wartości.

\section{Moduł wnioskowania rozmytego}
% ============================================================
Pierwszym etapem pracy systemu jest wnioskowanie rozmyte. Moduł operuje na
zbiorach rozmytych stworzonych na podstawie atrybutów \textit{ciężar},
\textit{długość} i \textit{długość życia}. Na ich podstawie tworzy nowy,
zbiorczy atrybut \textit{wielkość}. Wnioskowanie oparte jest na iloczynowej
t-normie.

Proces składa się z trzech etapów:
\begin{enumerate}
    \item \textbf{Rozmywanie} -- wartość liczbowa konwertowana jest na stopnie
          przynależności do zbiorów rozmytych za pomocą funkcji z tabeli~\ref{tab:fuzzy}.
    \item \textbf{Wnioskowanie} -- reguły rozmyte łączą przynależności
          poszczególnych cech. Przyjęta iloczynowa t-norma indukuje operatory:
          \begin{align}
              a \wedge b &= \top(a,b) = a \cdot b \\
              a \vee b   &= \bot(a,b) = a + b - a \cdot b
          \end{align}
          W implementacji koniunkcja realizowana jest przez predykat
          \code{t\_norma/3}, a alternatywa przez \code{s\_norma/3}.
    \item \textbf{Wyostrzanie} -- wartość wynikowa wybierana jest metodą maksimum
          (reguła rozmyta o najwyższym stopniu spełnienia wyznacza wartość
          atrybutu \textit{wielkość}):
          \begin{equation}
              \arg\max_{x \in U} \mu(x)
          \end{equation}
\end{enumerate}

\begin{lstlisting}[language=Prolog,caption={Reguła rozmyta i defuzyfikacja (\code{fuzzy.pl})}]
t_norma(A, B, V) :- V is A * B.
s_norma(A, B, V) :- V is A + B - (A * B).

fuzzy_rule(srednia, Ciezar, Dlugosc, _Zycie, V) :-
    ciezar(Ciezar, sredni, Vc),
    dlugosc(Dlugosc, sredni, Vd),
    t_norma(Vc, Vd, V).

calculate_wielkosc(Ciezar, Dlugosc, Zycie, Wielkosc) :-
    findall(V-W, fuzzy_rule(W, Ciezar, Dlugosc, Zycie, V), Results),
    sort(0, @>=, Results, [MaxV-MaxW | _]),
    ( MaxV > 0 -> Wielkosc = MaxW
    ; Wielkosc = niestandardowa
    ).
\end{lstlisting}

\noindent\textit{Uwaga implementacyjna:} reguły rozmyte zostały przyjęte
w uproszczonej postaci łączącej cechy \textit{ciężar} i \textit{długość}
(zgodnie z komentarzem w kodzie), ponieważ pełny zestaw reguł nie był
jednoznacznie określony w specyfikacji. Jeżeli żadna reguła nie zostaje
spełniona w stopniu większym od zera, atrybutowi nadawana jest wartość
\textit{niestandardowa}.

\subsection{Przykład liczbowy}
Prześledźmy działanie modułu na przykładzie \textit{kozy} o parametrach:
ciężar $= 60$~kg, długość $= 110$~cm, długość życia $= 15$~lat.

\paragraph{Krok 1 -- rozmywanie.}
Wartości liczbowe przekładane są na stopnie przynależności do zbiorów rozmytych
(tabela~\ref{tab:fuzzy}). Niezerowe przynależności wynoszą:
\begin{align*}
\text{ciężar }60: \quad
  &\mu_{\text{średni}} = \tfrac{100-60}{100-50} = 0{,}80, &
  &\mu_{\text{ciężki}} = \tfrac{60-50}{100-50} = 0{,}20, \\
\text{długość }110: \quad
  &\mu_{\text{średni}} = \tfrac{120-110}{120-50} \approx 0{,}14, &
  &\mu_{\text{duży}} = \tfrac{110-80}{200-80} = 0{,}25.
\end{align*}

\paragraph{Krok 2 -- wnioskowanie.}
Reguły rozmyte łączą przynależności iloczynową t-normą
($\top(a,b)=a\cdot b$). Spełnienie uzyskują dwie reguły:
\begin{align*}
\text{wielkość} = \text{średnia}:\quad & \mu_{\text{średni}}^{\text{ciężar}} \cdot \mu_{\text{średni}}^{\text{długość}} = 0{,}80 \cdot 0{,}14 \approx \mathbf{0{,}114}, \\
\text{wielkość} = \text{duża}:\quad & \mu_{\text{ciężki}}^{\text{ciężar}} \cdot \mu_{\text{duży}}^{\text{długość}} = 0{,}20 \cdot 0{,}25 = 0{,}050.
\end{align*}

\paragraph{Krok 3 -- wyostrzanie.}
Metodą maksimum wybierana jest reguła o najwyższym stopniu spełnienia:
\[
\arg\max\{0{,}114;\ 0{,}050\} \implies \text{wielkość} = \textbf{średnia}.
\]
Wynik ten został potwierdzony uruchomieniem predykatu
\code{calculate\_wielkosc(60.0, 110.0, 15.0, W)}, który zwraca
\code{W = srednia}.

\section{Moduł wnioskowania przybliżonego}
\label{sec:rough}
% ============================================================
Moduł wnioskowania przybliżonego (\code{rough.pl}) przeprowadza ostateczną
klasyfikację zwierzęcia. Na wejściu otrzymuje atrybuty warunkowe pochodzące od
użytkownika oraz jeden atrybut (\textit{wielkość}) wyznaczony przez moduł
wnioskowania rozmytego. Matematyczną podstawą modułu jest teoria zbiorów
przybliżonych.

Tablica decyzyjna to para $DT=(U, A \cup \{d\})$, gdzie $U$ to zbiór obiektów
(gatunków), $A$ to zbiór atrybutów warunkowych, a $d$ jest atrybutem decyzyjnym.
Zbiór atrybutów warunkowych zdefiniowany jest jako:

\begin{lstlisting}[language=Prolog]
all_attributes([wielkosc, dieta, srodowisko, skrzydla, nogi, udomowione,
                rozrod, futro, klimat, gromada, stadne, kontynent]).
\end{lstlisting}

Wartości atrybutów pobierane są jednolicie przez predykat \code{get\_attr/3},
który dla atrybutu \textit{wielkość} sięga do wyniku modułu rozmytego, a dla
pozostałych -- do bazy wiedzy. Przykładowy fragment tablicy decyzyjnej (dla
czytelności ograniczony do podzbioru atrybutów warunkowych oraz sześciu
obiektów) przedstawiono w tabeli~\ref{tab:dt}. Wartości atrybutu
\textit{wielkość} pochodzą z rzeczywistych obliczeń modułu rozmytego.

\label{tab:dt}

dodatkowe atrybuty (np. \textit{gromada}, \textit{środowisko}), co ilustruje
rolę pełnego zbioru atrybutów warunkowych.

\subsection{Relacja nierozróżnialności}
Relację nierozróżnialności generowaną przez zbiór atrybutów $B \subseteq A$
definiujemy następująco:
\[
\mathrm{IND}(B) = \{(x,y)\in U\times U : \forall_{a\in B}\; a(x)=a(y)\}
\]
Jest to relacja równoważności dzieląca $U$ na klasy abstrakcji obiektów
nierozróżnialnych. Klasa abstrakcji obiektu $x$:
\[
[x]_B = \{y\in U : (x,y)\in \mathrm{IND}(B)\}
\]
W implementacji odpowiadają im predykaty \code{indiscernible/3},
\code{equivalence\_class/4} oraz \code{all\_equivalence\_classes/3}.

\subsection{Przybliżenie dolne i górne}
Dla zbioru $X \subseteq U$ względem $B \subseteq A$ definiujemy:
\[
\underline{B}(X) = \bigcup\{Y\in U/\mathrm{IND}(B) : Y\subseteq X\}, \qquad
\overline{B}(X) = \bigcup\{Y\in U/\mathrm{IND}(B) : Y\cap X \ne \emptyset\}
\]
Realizują je predykaty \code{lower\_approximation/3} i
\code{upper\_approximation/3}.

\subsection{Niespójność}
Niespójność zachodzi, gdy dwa obiekty nierozróżnialne przez $A$ mają różne
decyzje. Zaproponowany system nie dopuszcza niespójności -- są one usuwane
\textbf{metodą jakościową}. Dla każdego obiektu wyznaczana jest dokładność
przybliżenia dolnego:
\[
\gamma_B(X) = \frac{|\underline{B}(X)|}{|U|}
\]
a usuwany jest obiekt o najniższej dokładności. Procedurę realizuje predykat
\code{remove\_inconsistencies/0} wraz z \code{find\_worst\_object/4}.

\subsection{Atrybuty niezbędne i jądro systemu}
Atrybut $a \in B$ nazywamy \textbf{zbędnym}, jeśli
$\mathrm{IND}(B) = \mathrm{IND}(B\setminus\{a\})$; w przeciwnym razie jest
\textbf{niezbędny}. Zbiór wszystkich atrybutów niezbędnych nazywamy jądrem
$\mathrm{CORE}(B)$. W systemie jądro wyznaczane jest dwiema metodami, które
służą do wzajemnej weryfikacji:
\begin{itemize}[noitemsep]
    \item jako zbiór atrybutów występujących pojedynczo we wpisach macierzy
          nierozróżnialności (\code{core\_from\_matrix/2}),
    \item przez bezpośrednie sprawdzenie warunku
          $\mathrm{IND}(B)\ne\mathrm{IND}(B\setminus\{a\})$
          (\code{core\_attributes\_verify/3}).
\end{itemize}

\subsection{Redukty}
Redukt to minimalny zbiór atrybutów zachowujący rozróżnialność obiektów --
musi zawierać jądro oraz mieć niepuste przecięcie z każdym niepustym wpisem
macierzy nierozróżnialności.

\subsubsection{Wyznaczanie reduktów z macierzy nierozróżnialności}
Macierz nierozróżnialności o wymiarach $n\times n$ ($n=|U|$) ma elementy:
\[
M_{ij} = \{a\in A : a(x_i)\ne a(x_j)\}
\]
Redukty wyznaczane są metodą \textbf{zachłanną}: algorytm startuje od jądra,
dodaje atrybuty pokrywające najwięcej niepokrytych wpisów
(\code{greedy\_cover/4}), a następnie próbuje usunąć atrybuty zbędne
(\code{minimize\_reduct/4}), tak by uzyskać zbiór minimalny.

\subsection{Reguły minimalne}
Przez regułę decyzyjną rozumiemy implikację postaci:
\[
a_1(x)=v_1 \wedge a_2(x)=v_2 \wedge \dots \wedge a_k(x)=v_k \implies dec(x)=d
\]
Generowanie reguł minimalnych przeprowadzono zgodnie z algorytmem
zaproponowanym przez Z.~Pawlaka i A.~Skowrona (\textit{A rough set approach to
decision rules generation}):
\begin{enumerate}[noitemsep]
    \item usunięcie niespójności z tablicy decyzyjnej $DT$ (metodą jakościową),
    \item utworzenie macierzy nierozróżnialności,
    \item dla każdego obiektu: utworzenie uogólnionej macierzy nierozróżnialności
          względem decyzji, jej minimalizacja (wyznaczenie reduktu) i zapis
          reguły decyzyjnej.
\end{enumerate}

Całość koordynuje predykat \code{generate\_reducts\_and\_rules/0}, a poszczególne
reguły zapisywane są dynamicznie jako fakty \code{dec\_rule/3}. Klasyfikacja
końcowa (\code{get\_decision/2}) odbywa się przez dopasowanie reguł do faktów
podanych przez użytkownika i wybór decyzji o największej liczbie pasujących reguł.

\begin{lstlisting}[language=Prolog,caption={Generowanie reguły dla obiektu (\code{rough.pl})}]
generate_rules_for_animal(Animals, Attrs, Animal) :-
    decision_discernibility_matrix(Animal, Animals, Attrs, Matrix),
    ( Matrix = [] -> true
    ; find_reduct(Matrix, Attrs, Reduct),
      create_rule(Animal, Reduct)
    ).
\end{lstlisting}

ograniczonej do czterech atrybutów warunkowych
$A=\{\textit{gromada}, \textit{środowisko}, \textit{skrzydła}, \textit{dieta}\}$.

\paragraph{Macierz nierozróżnialności względem decyzji dla $x_1$.}
Dla obiektu $x_1$ (lew) tworzymy wpisy $M_{1j}=\{a\in A: a(x_1)\ne a(x_j)\}$
porównując go z obiektami o innej decyzji:
\begin{align*}
M_{1,2} &= \{\textit{gromada}, \textit{środowisko}, \textit{skrzydła}\}, &
M_{1,3} &= \{\textit{gromada}, \textit{środowisko}\}, \\
M_{1,4} &= \{\textit{dieta}\}, &
M_{1,5} &= \{\textit{gromada}, \textit{środowisko}\}.
\end{align*}

\paragraph{Funkcja nierozróżnialności i jej minimalizacja.}
Funkcja przyjmuje postać koniunkcji alternatyw (CNF):
\[
f(x_1) = (\textit{gr}\vee\textit{śr}\vee\textit{sk}) \wedge (\textit{gr}\vee\textit{śr}) \wedge (\textit{dieta}) \wedge (\textit{gr}\vee\textit{śr}).
\]
Atrybut \textit{dieta} występuje samodzielnie -- należy do jądra i musi znaleźć
się w redukcie. Pozostałe klauzule pokrywa pojedynczy atrybut \textit{gromada}
(lub równoważnie \textit{środowisko}). Algorytm zachłanny
(\code{find\_reduct/3}) wybiera zatem redukt $\{\textit{dieta}, \textit{gromada}\}$.

\paragraph{Zapis reguły.}
Na podstawie reduktu i wartości atrybutów obiektu $x_1$ powstaje reguła:
\[
\textit{gromada}=\text{ssaki} \;\wedge\; \textit{dieta}=\text{mięsożerne}
\implies \text{lew}.
\]
W pełnym systemie (12 atrybutów warunkowych) dla lwa generowana jest analogiczna,
lecz dłuższa reguła oparta na innych atrybutach rozróżniających:
\code{dieta=miesozerne $\wedge$ kontynent=afryka $\wedge$ stadne=tak => lew}.

\section{Architektura Systemu -- Separacja Warstw}
\label{sec:arch}
% ============================================================
Projekt przestrzega zasady ścisłego rozdzielenia warstw, co jest kryterium
projektowym o szczególnym znaczeniu dla systemów z bazą wiedzy.

\subsection{Komponenty systemu}
System podzielony jest na cztery główne komponenty, każdy w osobnym module
Prologa:
\begin{enumerate}
    \item \textbf{Baza danych i wiedzy} (\code{kb.pl}) -- zawiera wyłącznie fakty
          i reguły dziedzinowe (zoologia). Nie zawiera logiki sterującej ani
          elementów interfejsu.
    \item \textbf{Mechanizm wnioskowania rozmytego} (\code{fuzzy.pl}) --
          implementuje czystą logikę rozmytą: funkcje przynależności, operatory
          t- i s-normy oraz wyznaczanie atrybutu \textit{wielkość}. Nie zna
          interfejsu użytkownika.
    \item \textbf{Moduł znajdowania reguł minimalnych} (\code{rough.pl}) --
          oddzielny komponent umożliwiający wydrukowanie atrybutów niezbędnych
          i reguł minimalnych na potrzeby analizy bazy wiedzy
          (\code{print\_indispensable/0}, \code{find\_minimal\_rules/0}).
    \item \textbf{Interfejs użytkownika} (\code{ui.pl}) -- odpowiada wyłącznie
          za komunikację z użytkownikiem i formatowanie wyników. Nie zawiera
          wiedzy dziedzinowej.
\end{enumerate}

\subsection{Wymagania dotyczące separacji}
Separacja warstw oznacza w szczególności:
\begin{itemize}[noitemsep]
    \item dane przedmiotowe nie pojawiają się w warstwie interfejsu,
    \item baza wiedzy jest oddzielona zarówno od interfejsu, jak i od logiki
          sterowania,
    \item konkluzje (reguły decyzyjne) konstruowane są dynamicznie w trakcie
          procesu wnioskowania (\code{dec\_rule/3} jako fakty dynamiczne) --
          nie są reprezentowane jako statyczne fakty w bazie wiedzy.
\end{itemize}

python gen_kb.py    # tworzy na nowo src/kb.pl

\section{Testy i Weryfikacja}
% ============================================================
Poprawność systemu weryfikowano na kilku poziomach:
\begin{itemize}[noitemsep]
    \item \textbf{Moduł rozmyty} -- wartości funkcji przynależności oraz wynik
          atrybutu \textit{wielkość} sprawdzano dla wybranych gatunków przez
          bezpośrednie wywołanie \code{calculate\_wielkosc/4} (np. koza
          $\rightarrow$ \textit{średnia}, lew $\rightarrow$ \textit{duża}, słoń
          $\rightarrow$ \textit{bardzo duża}).
    \item \textbf{Spójność bazy wiedzy} -- moduł analizy wykrywa i usuwa
          niespójności metodą jakościową; po tym etapie z~91 wygenerowanych
          reguł żadna nie prowadzi do sprzecznej decyzji.
    \item \textbf{Jądro systemu} -- wyznaczane jest dwiema niezależnymi metodami
          (z~macierzy nierozróżnialności oraz przez warunek
          $\mathrm{IND}(B)\ne\mathrm{IND}(B\setminus\{a\})$), które dają zgodny
          wynik, co potwierdza poprawność implementacji.
    \item \textbf{Dialog} -- przeprowadzono sesje identyfikacyjne dla gatunków
          z~różnych gromad, weryfikując, że system zbiega do jednego gatunku oraz
          że decyzja modułu rough set pokrywa się z~wynikiem dialogu.
\end{itemize}

\section{Ograniczenia i Kierunki Rozwoju}
% ============================================================
Zrealizowany system ma kilka znanych ograniczeń, które wyznaczają naturalne
kierunki dalszego rozwoju:
\begin{itemize}[noitemsep]
    \item \textbf{Uproszczone reguły rozmyte} -- atrybut \textit{wielkość}
          wyznaczany jest na podstawie sparowanych kategorii ciężaru i długości;
          dla nietypowych proporcji (np. długi, lecz lekki wąż) zwracana jest
          wartość \textit{niestandardowa}. Rozszerzenie zbioru reguł rozmytych
          poprawiłoby pokrycie takich przypadków.
    \item \textbf{Brak obsługi błędnych odpowiedzi} -- podanie wartości spoza
          listy opcji zawęża zbiór hipotez do pustego; warto dodać walidację
          i możliwość cofnięcia odpowiedzi.
    \item \textbf{Wagi reguł} -- obecnie wszystkie reguły mają wagę~1
          (\code{dec\_rule/3}); wprowadzenie wag opartych na pokryciu lub
          pewności umożliwiłoby ranking konkurencyjnych hipotez.
    \item \textbf{Interfejs} -- system działa w trybie tekstowym; możliwe jest
          opracowanie graficznego interfejsu użytkownika.
\end{itemize}

\section{Podsumowanie}
% ============================================================
Zrealizowany system ekspertowy do identyfikacji gatunków zwierząt stanowi
praktyczną implementację koncepcji wnioskowania regułowego wzbogaconego
o mechanizmy logiki rozmytej i teorii zbiorów przybliżonych. Kluczowymi
elementami wyróżniającymi system są:
\begin{itemize}[noitemsep]
    \item \textbf{Wnioskowanie w warunkach niepewności} -- dzięki funkcjom
          przynależności system radzi sobie z rozmytymi parametrami i płynnymi
          granicami między kategoriami,
    \item \textbf{Dynamiczny, kontekstowy dialog} -- liczba pytań jest
          minimalizowana przez wybór atrybutów o najniższej entropii,
    \item \textbf{Moduł analizy minimalnej} -- narzędzie do weryfikacji
          i optymalizacji bazy wiedzy, wykrywające niespójności, atrybuty
          niezbędne i redukty,
    \item \textbf{Czysta separacja warstw} -- architektura gwarantująca
          niezależność bazy wiedzy, silnika wnioskowania, modułu analizy
          i interfejsu.
\end{itemize}

Zastosowanie języka Prolog jako platformy implementacyjnej pozwoliło w naturalny
sposób wyrazić reguły produkcji i mechanizm wnioskowania wstecznego, a rozszerzenie
o operatory rozmyte i zbiory przybliżone umożliwiło obsługę rzeczywistych,
niedoskonałych danych wejściowych.
